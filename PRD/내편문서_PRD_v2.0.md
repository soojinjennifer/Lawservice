## 1. 서비스 개요

### 1.1 제품 비전

법률 지식이 없는 일반인도 변호사 사무실을 방문하지 않고, 억울한 일을 자신의 언어로 입력하는 것만으로 형식과 내용을 갖춘 법률 문서 초안을 즉시 받아볼 수 있도록 한다. 나아가 업로드한 증거에서 AI가 사건 경위를 시간순으로 정리하고, 나의 주장과 증거를 연결하며, 상대방 문서의 핵심 주장과 반박 쟁점을 분석함으로써 변호사 상담 전 법률 문서 준비의 효율을 크게 높인다.

**슬로건**: "억울한 일을 법률문서의 언어로 정리해드립니다."

**서비스명**: 내편문서

**AI 엔진**: Claude (claude-sonnet-4-6)

### 1.2 배경 및 문제 정의

일반 소송 의뢰인이 겪는 3가지 핵심 어려움:

1. 법률 문서의 형식(서식)을 알지 못해 어디서부터 작성해야 할지 막막하다.
2. 변호사 상담·문서 작성 비용이 부담스럽고 진입 장벽이 높다.
3. 인터넷의 양식을 받아도 자신의 사안에 맞게 사실관계를 구성하기 어렵다.

본 서비스는 문서 종류별 표준 서식 가이드와 AI 생성을 결합해 이 문제를 해결한다.

### 1.3 제품 목적

- 문서 종류 선택 → 정보 입력 → 증거 업로드·정리 → AI 초안 생성 → 수정 → 저장의 확장된 흐름 제공
- 사건 타임라인 자동 정리, 주장-증거 매핑, 상대방 주장 반박 목록 생성을 통해 단순 초안 생성을 넘어 사건 준비 도구로 기능
- 건별 결제 구조로 부담 없이 필요한 문서만 구매하고, 단계적으로 고급 기능 이용 지원

### 1.4 범위 (In / Out of Scope)

**포함 범위 (In Scope)**
- 문서 3종 활성 초안 생성·수정·저장(.docx)
- 회원가입/로그인/마이페이지
- 건별 결제 체계 (첫 내용증명 1건 무료)
- 문서 종류별 도움말·발송 가이드
- 증거 업로드 및 AI 자동 추출
- 홈 대시보드 (로그인 사용자 전용)
- 상대방 주장 분석·반박 목록 (반박문 전용)
- 위험 문장 검사

**제외 범위 (Out of Scope)**
- 실제 법원 전자소송 시스템 연동·자동 제출
- 변호사 1:1 상담·수임 매칭
- 문서의 법적 효력·정확성 보장
- 타 언어(영문 등) 문서 생성 (v3.0 과제)
- 기업·구독 월정액 플랜 (추후 출시)

> **법적 고지**: 본 서비스가 생성하는 모든 문서는 참고용 초안이며, 법적 효력이나 정확성을 보장하지 않는다. 실제 제출 전 전문가 검토를 권고한다.

## 3. 목표 및 성공지표

### 3.1 제품 목표

- 일반 사용자가 10분 이내에 첫 문서 초안을 완성한다.
- AI 초안을 그대로/약간의 수정만으로 사용하는 비율을 높인다.
- 무료 체험 사용자를 유료 건별 결제로 전환한다.

### 3.2 핵심 성공지표 (KPI)

| 지표 | 정의 | 목표 |
|---|---|---|
| 초안 완성률 | 문서 생성 시작 → 저장까지 완료한 세션 비율 | ≥ 60% |
| 평균 소요 시간 | 정보 입력 시작 → .docx 저장까지 시간 | ≤ 10분 |
| 수정 횟수 | 초안 1건당 평균 대화형 수정 요청 수 | ≤ 3회 |
| 유료 전환율 | 무료 체험 → 건별 결제 전환 비율 | ≥ 8% |
| 재방문율 | 30일 내 재사용 사용자 비율 | ≥ 35% |
| 문서 만족도 | 저장 후 만족도(별점) 평균 | ≥ 4.2 / 5 |

## 5. 서비스 전체 플로우

### 5.1 상위 플로우

```
[비로그인]  홈(공감 카피) → 문서 종류 클릭 / "지금 내 억울함 정리하기"
            → 회원가입(무료 체험 1건) → 문서 생성

[로그인]    로그인 → 홈=대시보드(사건 진행 카드/다음 단계 안내)
            → "이어서" 또는 "다음 추천 문서" → 문서 생성

[문서 생성] 1. 문서 정보 입력 (증거 업로드 포함)
            → 2. 초안 미리보기
            → 3. 내용 수정 및 저장
            (반박문일 때) 1단계에 상대방 문서 분석·반박 목록 포함
            저장 전/결제 필요 시 → 결제 Modal → 생성 진입
```

| 단계 | 화면 | 설명 |
|---|---|---|
| ① | 회원가입 | 이메일/소셜 가입, 약관 동의, 이메일 인증. 가입 시 무료 체험 자격 자동 부여 |
| ② | 로그인 | 이메일/비밀번호 또는 소셜 로그인, 비밀번호 재설정 |
| ③ | 홈(랜딩) | 서비스 소개·활용 방법·요금제 안내. 문서 생성 진입점. 로그인 시 대시보드 표시 |
| ④ | 문서 생성 | 문서 종류 선택 후 3단계 프로세스 탭 진행 |

### 5.2 문서 생성 하위 프로세스

문서 생성은 상단의 StepIndicator로 3단계를 표시한다.

| 탭 | 단계 | 주요 동작 |
|---|---|---|
| 1 | 문서 정보 입력 | 문서 종류 선택, 발신/수신인, 사건 표시, 시간순 사건 경위, 증거 업로드, 반박문은 상대방 분석 포함 → 「초안 생성」 |
| 2 | 초안 미리보기 | AI 생성 초안 확인. 좌: 문서(스크롤), 우: 증거 목록 + 증거별 다운로드 버튼 |
| 3 | 내용 수정 및 저장 | 대화형 수정 요청(3회 포함), 위험 문장 검사, 최종본 .docx 저장 |

**탭 이동 규칙**:
- 1단계 필수 입력(사건 경위)이 채워져야 2단계로 진행 가능
- 2단계에서 초안이 생성되어야 3단계(수정·저장) 활성화
- 이전 탭으로 되돌아가 내용 수정 가능 (StepIndicator clickable: true)

**임시 저장 규칙**:
- sessionStorage 기반 (브라우저 탭이 열린 동안 유지, 탭·창 닫으면 자동 삭제)
- 저장 대상: 문서 종류·발신/수신인·사건 표시·타임라인·사건 경위·요구사항·**증거 파일 목록(업로드 완료분)**
- 재진입 시 자동 복원 및 "임시 저장 복원됨" 토스트 표시
- 증거 파일 목록 하단에 "증거 파일은 임시로 보관되며 서버에 저장되지 않습니다." 안내 표시

## 6. 정보구조 및 GNB

### 6.1 글로벌 내비게이션 (GNB)

| 메뉴 | 역할 / 연결 화면 |
|---|---|
| 홈 | 랜딩 페이지. 로그인 시 대시보드로 분기 |
| 문서 생성 | 문서 종류 선택 후 3단계 프로세스 진입 |
| 법률 문서 도움말 | 문서 종류별 설명, 작성 팁, 우체국 내용증명 발송 가이드 |
| 마이페이지 | 나의 문서, 결제 내역 |
| FAQ | 자주 묻는 질문, 결제·환불·문서 효력 관련 안내 |
| (로그인 사용자) | "안녕하세요, OOO님" + 로그아웃 버튼, 누적 N건 / 결제액 칩 표시 |

**GNB variant**:
- `guest`: 로그인 + 무료로 시작하기 버튼
- `authed`: 크레딧 칩("누적 N건") + 알림/도움말 + "안녕하세요, OOO님" + 로그아웃 (아바타·드롭다운 제거)

### 6.2 사이트맵

```
홈 (랜딩)
  ├─ 서비스 소개 / 공감 카피 섹션
  ├─ 활용 방법
  ├─ 요금제 (건별 결제)
  └─ 대시보드 (로그인 시) — 사건 진행 카드 / 생성 문서 바로가기 / 진행 문의 / 다음 단계 안내

  ├─ 1단계: 정보 입력 (문서 종류별 동적 폼, 증거 업로드)
  ├─ 2단계: 초안 미리보기
  └─ 3단계: 수정 및 저장

법률 문서 도움말
  ├─ 내용증명 (5단계 가이드 구현 완료)
  ├─ 준비서면 (추가 예정)
  └─ 상대방 반박문 (추가 예정)

마이페이지
  ├─ 나의 문서 (목록 / 보기 / 재다운로드 / 이어서 수정 / 삭제)
  └─ 결제 내역

FAQ

인증
  ├─ 회원가입
  ├─ 로그인
  └─ 비밀번호 재설정
```

## 7. 사용자 시나리오

### 7.1 시나리오 A — 내용증명 무료 첫 체험

- **사용자**: 김미수 (자영업자, 미수금 독촉 목적)
- **상황**: 처음 방문, 비로그인 상태
- **행동 흐름**:
  1. 홈에서 공감 섹션 읽고 "지금 내 억울함 정리하기" CTA 클릭
  2. 회원가입 후 무료 체험 안내 확인
  3. 문서 생성 → 내용증명 선택
  4. 발신인·수신인·사건 경위·요구사항 입력
  5. 「초안 생성하기」 클릭 → 로딩 → 초안 미리보기
  6. 내용 검토 후 「docx 받기」 클릭
  7. 다운로드 완료 → Toast "저장완료" 표시 → 마이페이지 이동

    - 수정된 텍스트 부분에 노란색 하이라이트(`#FFF9C4`) 배경 표시

#### 미리보기·수정 화면 레이아웃 (v2.0 확정, Figma 56:8418 기준)

> 아래 레이아웃은 Figma 56:8418을 기준으로 구현 완료됨. 우측 패널에 증거 목록이 아닌 AI 어시스턴트만 배치하고, 증거 목록은 2열 아래 전폭 섹션으로 분리된 것이 최종 확정안.

- **2단계 미리보기 레이아웃** (`create-edit-layout` 2열 그리드):
  - 좌측: 문서 영역 — 스크롤 적용 (`max-height: calc(100vh - 320px)`)
  - 우측: AI 어시스턴트(채팅) — **보이되 비활성화** (반투명 오버레이 + "다음 단계에서 수정할 수 있어요" 안내)
  - 2열 아래 전폭: 증거 자료 목록 (`create-evidence-row` — 카드 그리드, 읽기 전용)
  - 하단 버튼 (일관 배치, `create-bottom-actions`): [이전] [다시 생성] [docx 받기] [다음]
- **3단계 수정 화면 레이아웃** (미리보기와 동일 2열):
  - 좌측: 문서 영역 (스크롤)
  - 우측: AI 어시스턴트(채팅) — **활성화** (대화형 수정 요청)
  - 2열 아래 전폭: 증거 자료 목록 (`create-evidence-row` — 카드 그리드, 읽기 전용) — **2단계와 동일하게 표시**
  - 하단 버튼: [이전] [다시 생성] [docx 받기] [완료]
  - "완료" 클릭 시: 확인 모달 ("작성이 완료되었습니다. 파일을 다운로드 하시겠습니까?" + [취소]/[docx 다운로드])
  - 다운로드 → "저장완료" Toast → 마이페이지 이동

**수정 내용 하이라이트 공통 규칙 (v2.1 추가)**:

## 13. 데이터 및 상태 정의

### 13.1 주요 데이터 객체

| 객체 | 주요 속성 |
|---|---|
| User (사용자) | id, 이메일, 이름, 비밀번호(해시), 가입일, 무료체험사용여부, 마케팅동의 |
| UsageQuota (사용량) | user_id, 문서종류별 무료체험 잔여, 문서별 수정 잔여 횟수 |
| Document (문서) | id, user_id, 문서종류, 제목, 입력값, 초안본문, 상태, 생성일/수정일 |
| CaseTimeline (경위) | document_id, 항목리스트[{시간, 사건내용}] |
| Evidence (증거) | document_id, 파일명, 증거번호, 자료명, 입증취지, AI추출결과, 업로드일 |
| Payment (결제) | user_id, 문서종류, 금액, 일시, 상태(성공/실패/환불) |

### 13.2 문서 상태 전이

```
작성중 (입력) → 초안생성됨 → 수정중 → 저장완료 → 발송완료
                                           ↑
                                      분석중 (반박문 상대방 분석 시)
```

각 상태는 프로세스 탭 단계와 매핑되며, 임시 저장으로 중단·재개 가능.

- `작성중`: 1단계 탭, 정보 입력 중
- `초안생성됨`: 2단계 탭, AI 초안 생성 완료
- `수정중`: 3단계 탭, 대화형 수정 진행 중
- `저장완료`: .docx 다운로드 완료
- `발송완료`: 사용자가 수동으로 상태 업데이트 (대시보드 체크박스)

| 파일명 한글 깨짐 | Content-Disposition의 filename*(UTF-8) 기준으로 처리 |
| 증거 파일 중복 | 자동으로 `_2`, `_3` 순번 부여 |

## 16. 제약 조건 및 가정

### 16.1 기술 제약

- 빌드 도구 없이 CDN(Babel standalone)으로 React 운용 — 복잡한 번들 최적화 제약
- sessionStorage 기반 임시 저장 — 브라우저 종료 시 증거 파일 포함 모든 임시 데이터 소멸
- AI 생성은 Claude API 의존 — Claude 서비스 장애 시 전체 문서 생성 기능 중단

### 16.2 비즈니스 제약

- 현재 개인 사업자(주식회사 더그라운드모여) 운영 — 대규모 인프라 비용 제약
- 법률 서비스 특성상 AI 한계 고지 필수

### 16.3 가정

- 사용자는 본인 사안의 사실관계를 직접 입력할 수 있다
- 사용자는 증거 파일을 본인 기기에서 업로드할 수 있다
- 결제는 국내 카드·카카오페이 등 국내 결제 수단만 지원

### 16.4 위험 및 완화 방안

| 위험 | 완화 방안 |
|---|---|
| 생성 문서의 정확성·법적 효력에 대한 사용자 오인 | 고지 문구 + 전문가 검토 권고 LegalNoticeBox 전 화면 표시 |
| 민감한 사건 정보 처리 | 보관·삭제·암호화 정책 수립, 개인정보처리방침 준수 |
| AI 생성 품질 편차 | 문서 종류별 참고문서(서식 가이드) 기반 프롬프트 엔지니어링 |
| 결제 실패 | 재시도 안내, 결제 실패 시 이전 무료 체험 상태 유지 |

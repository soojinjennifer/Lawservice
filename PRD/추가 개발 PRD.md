# 추가 개발 PRD — v1.2 대비 변경점

> 기준 문서: `AI_법률문서_자동화_서비스_PRD_v1.2.docx` (문서 버전 v1.2, 최종 수정 2026-06-02)
> 이 문서는 v1.2 PRD에서 **바뀐 부분만** 정리한다. 각 항목은 〈v1.2 근거〉와 〈변경점〉으로 구성한다.

---

## 1. 증거 자료 명기 및 저장

- **v1.2 근거**: FR-17 "증거 파일 업로드 — 카카오톡 캡처·문자·계약서/PDF·입금내역 지원, AI 날짜·금액 자동 추출, 타임라인 자동 배치" (P1, v2.0 과제). §7.1 공통 출력 규칙은 "문서 하단에 「참고용 초안」 고지 문구 삽입"까지만 규정.
- **v1.2 원문 요지**: 증거는 업로드 후 AI가 날짜를 추출해 **타임라인에 배치**하는 수준. 증거 파일의 저장 위치·파일명 규칙·문서 내 증거목록 명기·다운로드 UI는 정의되지 않음.
- **변경점(v1.2 → 현재)**:
  - 업로드 증거를 `EVIDENCE_NAMING_GUIDE.md` 규칙으로 **자동 명명**하여 `/증거파일` 폴더에 저장(중복 시 `_2…`).
  - 증거번호 체계: 준비서면 `갑 제N호증`, 상대방 반박문 `을 제N호증`, 내용증명 `첨부 제N호`.
  - 생성된 초안 **문서 하단에 「입 증 방 법」 섹션을 서버에서 결정적으로 자동 생성**(증거번호·자료명·입증취지 순).
  - **초안 미리보기 우측에 증거 목록 + 증거별 다운로드 버튼** 배치.
  - 파일명은 가이드 포맷에서 **날짜를 제외**한 `[증거번호]_[자료명]_[간단설명].ext`로 적용(예: `갑제1호증_카카오톡대화_지급약속.png`).

## 2. 임시 저장에 증거 파일 포함

- **v1.2 근거**: FR-15(임시 저장), §6.4 탭1 동작 — "sessionStorage에 모든 입력값(문서 종류·발신/수신인·사건 표시·타임라인·사건 경위·요구사항)을 저장".
- **v1.2 원문 요지**: 임시 저장 대상에 **증거 파일은 포함되지 않음**.
- **변경점**: 임시 저장 시 **증거 파일 목록(업로드 완료분)도 함께 보관·복원**. 단 서버가 아닌 **sessionStorage에 임시 보관**(브라우저 종료 시 소멸). 증거 목록 하단에 "증거 파일은 임시로 보관되며 서버에 저장되지 않습니다." 안내 표시.

## 3. 다운로드 문서 고지 문구

- **v1.2 근거**: §7.1 "문서 하단에 「참고용 초안」 고지 문구 삽입", §10 법적 고지 "모든 출력 문서·화면에 '참고용 초안, 법적 효력 미보장' 고지 노출".
- **v1.2 원문 요지**: 고지 문구를 **출력 문서(다운로드 포함)와 화면 모두**에 표기.
- **변경점(상충 → 갱신)**: 고지 문구("본 문서는 법률적 참고용 초안이며…", "※ 본 문서는 AI(Claude)가 자동 생성한 참고용 초안입니다…")를 **다운로드 .docx에는 표시하지 않고, 초안 미리보기 화면에만** 표시. (다운로드 문서의 깔끔한 제출을 위해 v1.2의 "모든 출력 문서" 규정을 화면 한정으로 조정)

## 4. 초안 미리보기·수정 화면 레이아웃

- **v1.2 근거**: §6.4 탭2(초안 미리보기) "「이전(정보 수정)」 및 「다음(수정·저장)」 이동 버튼", 탭3(내용 수정 및 저장) "대화형 수정 요청 입력창 + 저장 형식 선택".
- **v1.2 원문 요지**: 탭2/탭3의 구체적 레이아웃(스크롤·버튼 위치·증거/AI 배치)은 미규정.
- **변경점**:
  - 미리보기·수정 화면 모두 **내용이 길어도 한 화면에 들어오고, 문서 영역에 스크롤** 적용.
  - **수정 화면도 미리보기와 동일한 레이아웃**: 좌측 문서(스크롤), 우측에 **증거 자료 목록 → 그 아래 AI 어시스턴트(채팅)** 배치.
  - 미리보기·수정 화면 하단에 **이전 / 다시 생성 / docx 받기 / 다음** 버튼을 일관 배치(상단 중복 버튼 제거).
  - 수정 화면의 "다음"은 **"완료"**: 확인 모달("작성이 완료되었습니다. 파일을 다운로드 하시겠습니까?" + [취소]/[docx 다운로드]) → 다운로드 → "저장완료" 토스트 → 마이페이지 이동.

## 5. 내용증명 상투 문구 제거

- **v1.2 근거**: §9.1 "문서 생성 시 /참고문서 파일을 참고하여 초안 생성"(내용증명→/내용증명 폴더). v1.2 본문에는 특정 인사 문구 규정 없음.
- **v1.2 원문 요지**: 내용증명 서식은 참고문서 기반. (※ "1. 귀사의 무궁한 발전을 기원합니다."는 v1.2가 아니라 서식 가이드 템플릿에 들어 있던 상투 문구)
- **변경점**: 내용증명 초안 생성 시 **상투 인사 문구("귀사의 무궁한 발전을 기원합니다" 등)를 넣지 않음**(서식 가이드에서 제거 + 생성 프롬프트에 부정 지시 추가).

## 6. 초안문서 다운로드 파일명

- **v1.2 근거**: FR-09 ".docx / .txt 파일 저장 및 다운로드".
- **v1.2 원문 요지**: 파일명 형식은 미규정.
- **변경점**: 다운로드 파일명을 **`문서종류명_[번호].docx`** 형식으로 정리. [번호]는 **날짜+시간**(예: `내용증명_20260614_0146.docx`). 한글 파일명이 깨지지 않도록 `Content-Disposition`의 `filename*`(UTF-8) 기준으로 처리.

1. 개요


1.1 제품 비전

법률 지식이 없는 일반인도 변호사 사무실을 방문하지 않고, 억울한 일을 자신의 언어로 입력하는 것만으로 형식과 내용을 갖춘 법률 문서 초안을 즉시 받아볼 수 있도록 한다. 나아가 업로드한 증거에서 AI가 사건 경위를 시간순으로 정리하고, 나의 주장과 증거를 연결하며, 상대방 문서의 핵심 주장과 반박 쟁점을 분석함으로써 변호사 상담 전 법률 문서 준비의 효율을 크게 높인다. 본 서비스(내편문서)는 AI(Claude)를 활용하여 "억울한 일을 법률문서의 언어로 정리해드립니다."


1.2 배경 및 문제 정의

일반 소송 의뢰인은 다음과 같은 어려움을 겪는다.


법률 문서의 형식(서식)을 알지 못해 어디서부터 작성해야 할지 막막하다.


변호사 상담·문서 작성 비용이 부담스럽고, 간단한 사안에도 진입 장벽이 높다.


인터넷의 양식을 받아도 자신의 사안에 맞게 사실관계를 구성하기 어렵다.

본 서비스는 문서 종류별 표준 서식 가이드와 AI 생성을 결합해 이 문제를 해결한다.


1.3 제품 목적


문서 종류 선택 → 정보 입력 → 증거 업로드·정리 → AI 초안 생성 → 수정 → 저장의 확장된 흐름을 제공한다.


사건 타임라인 자동 정리, 주장-증거 매핑, 상대방 주장 반박 목록 생성을 통해 단순 초안 생성을 넘어 사건 준비 도구로 기능한다.


건별 결제 구조로 부담 없이 필요한 문서만 구매하고, 단계적으로 고급 기능을 이용할 수 있도록 지원한다.

4.2 문서 생성 하위 프로세스 (프로세스 탭)

문서 생성은 상단의 프로세스 탭(스텝 인디케이터)으로 구성되며, 사용자는 순서대로 진행한다. 각 단계는 완료 시 다음 탭이 활성화되고, 이전 탭으로 되돌아가 내용을 수정할 수 있다.

1. 문서 정보 입력  →  2. 초안 미리보기  →  3. 내용 수정 및 저장

탭

단계

주요 동작

1

문서 정보 입력

문서 종류 선택, 발신/수신인, 사건 표시, 시간순 사건 경위, 사건 경위·요구사항 입력 → 「초안 생성」

2

초안 미리보기

AI가 생성한 초안을 서식이 적용된 형태로 확인. 로딩 인디케이터 표시

3

내용 수정 및 저장

대화형 수정 요청으로 초안 보완(요금제별 횟수 제한), 최종본을 .docx/.txt로 저장

탭 이동 규칙: 1단계 필수 입력(사건 경위)이 채워져야 2단계로 진행 가능하며, 2단계에서 초안이 생성되어야 3단계(수정·저장)가 활성화된다. 임시 저장은 sessionStorage 기반으로 구현되며, 브라우저 탭이 열려 있는 동안 입력 내용을 유지한다. 탭·창을 닫으면 자동 삭제되고, 재진입 시 자동 복원 및 "임시 저장 복원됨" 표시가 나타난다.

비로그인·로그인 사용자 모두의 진입점이며, 로그인 상태에 따라 콘텐츠가 분기된다.


① AI 법률 문서 자동화 서비스 소개 (비로그인 / 로그인 공통)


서비스 정의 문구: "여러분의 언어로 털어놓은 억울한 일을 AI가 참고용 법률 문서 초안을 생성하고, 제출 전 확인 항목을 안내해 드립니다."


생성 가능한 문서: 내용증명 / 상대방 답변·반박문 / 준비서면 초안 / 소견서 / 항소이유서 / 계약서(오픈 예정) — 각 문서 카드 클릭 시 해당 문서 생성으로 바로 진입.


공감 문구 섹션: "억울한 일이 있거나 해결이 안 되는 일이 있으면 답답하기만 하셨죠?", "무조건 변호사를 찾아가기도 겁나고, 무슨 이야기를 어떻게 해야 하는지 막막하셨죠?", "상대방이 이렇게 답했는데 이제 어떡하지? 매번 변호사 상담하기도 난감하셨죠?" — 공감 문구 이후 CTA 버튼 「지금 내 억울함 정리하기」.


② 활용 방법


3단계 사용법: 정보·증거 입력 → 주장 구성·상대방 분석 → 초안 생성·수정·저장.


서비스 핵심 설명 문구: "증거를 올리면 AI가 사건 경위를 시간순으로 정리합니다.", "법률문서에서 제일 중요한 건 문장의 수려함이 아니라, 주장과 증거의 연결입니다.", "상대방 문서를 올려주세요. 핵심 주장·인정하면 안 되는 부분·반박해야 할 쟁점·필요한 증거를 정리해드립니다."


문서 종류별 활용 사례: 미수금 독촉(내용증명), 상대 답변서 분석(반박문), 항소 준비(항소이유서) 등.


③ 요금제 (건별 결제)


내용증명 1건 9,900원 / 준비서면 초안 1건 49,000원 / 상대방 문서 분석+반박서 초안 99,000원.


「문서 구매하기」 CTA → 결제 후 해당 문서 생성 화면으로 이동.

사건 진행 프로세스 카드: 진행 중인 사건의 현황(문서 종류·제목·생성일·상태)을 카드 형태로 표시.

진행 과정 문의 카드 (체크박스): 내용증명 — "상대방이 답변을 했나요?", "내용증명이 반송되었나요?". 상대방 답변·반박문/준비서면 — "상대방이 답변을 했나요?", "합의 제안을 받았나요?", "더 심한 주장을 보냈나요?".

9. 기능 요구사항 (Functional Requirements)


9.1 기술 구현 현황 (v1.1 기준)
현재 구현된 기술 스택 및 아키텍처 사항은 다음과 같다.


프론트엔드 / 백엔드 완전 분리: Flask 서버(server.py) + 정적 파일(HTML·CSS·JS). React + JSX는 CDN(Babel standalone)으로 처리하며 빌드 도구 없이 운용.


API 키 보안: Anthropic API 키는 서버의 .env 파일에만 저장하고, load_dotenv(override=True)로 환경변수 우선 적용. 브라우저에 키가 노출되지 않음.


Claude 모델: claude-sonnet-4-6 (max_tokens 3500). 초안 생성(/api/generate)·수정(/api/revise)·docx 다운로드(/api/download_docx) 3개 엔드포인트.


DOCX 출력: python-docx로 제목(맑은 고딕·굵게·18pt)·본문(바탕체·11pt) 서식 자동 적용. 정렬 규칙(가운데=섹션 제목, 우측=날짜·서명, 좌측=본문) 키워드 매칭으로 자동화.


임시 저장: sessionStorage 기반. 탭 열린 동안 유지, 탭·창 닫으면 자동 삭제.


디자인 시스템: Microsoft Fluent 2 Web 토큰(Colors 32개·Radius 9개·Spacing 13개·Typography 11개, 총 65개)을 Figma Variable Collection으로 구축. DocType Card 컴포넌트의 fill/stroke 99% 변수 바인딩.

13.2 가정 및 위험


13.3 향후 기능 과제 (수정PRD초안 기반)


증거 파일 업로드: 카카오톡 캡처·문자 내용·계약서·PDF·입금내역 업로드 및 AI 날짜 자동 추출.


주장·증거 매핑: AI가 나의 주장별 필요 증거 목록을 추출하고 현재 상태(있음/필요/일부 부족)를 표시.


상대방 주장 분석: 상대방 문서 업로드 시 핵심 주장·인정 불가 부분·반박 쟁점을 자동 추출하고 반박 목록 생성.


위험 문장 검사: 과격하거나 법률에 맞지 않는 문장을 AI가 검사하고 수정 제안.


홈 대시보드(로그인 상태): 사건 진행 프로세스 카드, 이전 생성 문서 목록, 다음 가능한 프로세스 안내.


소장(訴狀) 문서 생성 추가.


영문 문서 생성 지원.


가정: 사용자는 본인 사안의 사실관계를 직접 입력할 수 있다.


위험: 생성 문서의 정확성·법적 효력에 대한 사용자 오인 → 고지 문구·전문가 검토 권고로 완화.


위험: 민감한 사건 정보 처리 → 보관/삭제·암호화 정책으로 완화.




※ 본 PRD는 v1.2이며, 수정PRD초안(내편문서)의 신규 기능·요금제·서비스 비전을 통합 반영합니다. 이해관계자 검토에 따라 변경될 수 있습니다.

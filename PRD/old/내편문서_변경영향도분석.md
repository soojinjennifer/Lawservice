# 내편문서 — 변경 영향도 분석 & 컴포넌트 구조 계획
> 1차 MVP("AI 법률 문서 자동화 서비스") → 2차 PRD v1.2("내편문서") 전환
> 기준: 기존 구조 최대 유지 · Fluent 2 Web Design System 적용 · Figma/Claude Code 재사용 가능 구조
> 상태: **분석/계획 단계 (디자인 착수 전, 승인 대기)**

## 0. 현재까지 반영된 부분 (진행 중, 승인 시 정리 예정)
전환 작업 일부가 이미 코드에 들어가 있습니다. 아래 분석의 "수정" 항목과 일치합니다.
- 서비스명 `내편문서` 전역 치환, 슬로건 "억울한 일을 법률문서의 언어로 정리해드립니다"
- 문서 종류 6종 재정의(`상대방 반박문` 추가), 건별 가격 필드 추가
- 홈 공감 카피 섹션 + 건별 결제 요금제 + 로그인 대시보드(`DashboardScreen`) 신규
- 문서 생성에 증거 업로드 / 상대방 분석·반박 목록 블록 추가
- 라우터에 `/dashboard` 추가, 로그인 → 대시보드 진입

## 1. 화면 영향도 분석

### ✅ 유지할 화면 (구조·흐름 그대로, 문구/브랜드만 치환)
| 화면 | 유지 이유 |
|---|---|
| 로그인 (`LoginScreen`) | PRD §6.2 요건 동일 (이메일/소셜/상태유지/비번재설정/잠금) |
| 회원가입 (`AuthScreen`) | PRD §6.1 요건 동일 (약관·소셜·이메일 인증) |
| 비밀번호 찾기 4단계 (`ResetPasswordScreen`) | 변경 없음 |
| 법률 문서 도움말 (`HelpScreen`) | PRD §6.5 내용증명 5단계 Closable Panel 그대로 |
| 마이페이지·나의 문서 (`MyDocsScreen`) | PRD §6.6 ① 목록/재다운로드/이어서 수정 동일 |
| FAQ (`FAQScreen`) | PRD §6.7 동일 (문구만 건별 결제 기준으로) |

### ✏️ 수정할 화면
| 화면 | 2차 PRD 변경점 |
|---|---|
| **홈 (`HomeScreen`)** | ① 공감 카피 섹션 + CTA "지금 내 억울함 정리하기" 신규 ② 문서 5→6종(반박문) ③ 활용법을 "증거 입력→주장·상대방 분석→생성" 으로 ④ 요금제 = **구독→건별 결제** |
| **문서 생성 (`CreateScreen`)** | ① 문서 종류에 `상대방 반박문` 추가(활성 3종) ② 증거 업로드(AI 날짜 추출) 블록 ③ 주장-증거 매핑 / 상대방 주장 분석·반박 목록(체크박스) ④ 위험 문장 검사 **버튼**(FR-20) ⑤ "현재 플랜"→"건별 결제" 표기 |
| **요금제 (`SubscriptionScreen`)** | 월 구독 → **건별 결제·구매 내역·잔여 무료체험(1건/수정 3회)** 으로 재편 |

### ➕ 새로 추가할 화면
| 화면 | 근거 |
|---|---|
| **홈 대시보드 (`DashboardScreen`)** | PRD §6.3 ④ / FR-16 — 로그인 시 홈에 사건 진행 카드·생성 문서·진행 과정 체크박스·다음 단계 안내 |
| (선택) **결제 모달 플로우** | FR-21 건별 결제 — 별도 화면보다 Modal 컴포넌트 인스턴스로 처리 권장 |

### 🔀 삭제/통합할 UI
| 대상 | 처리 |
|---|---|
| Basic/Standard/Pro **월 구독 3단 카드** | 건별 결제 카드로 통합 (구독은 PRD상 "기업·반복 사용자용 추후" → 화면 제거) |
| 문서 종류 5→3종 | 활성 3종(내용증명·준비서면·반박문) / 소견서·항소이유서 = **v2.0 예정** 배지 / 계약서 = Coming Soon |
| "내용증명+발송가이드 29,000원" 상품 | **삭제** (단일 내용증명 19,900원으로 통합) |
| GNB의 "구독 플랜·수정 3회" 크레딧 표기 | "무료 1건 / 건별 결제" 로 통합 |

| 소견서 / 항소이유서 1건 | 99,000원 | **v2.0 출시 예정** |

## 2. 변경되는 사용자 흐름
```
[비로그인]  홈(공감 카피) → 문서 종류 클릭 / "지금 내 억울함 정리하기"
            → 회원가입(무료 체험 1건) → 문서 생성

[로그인]    로그인 → 홈=대시보드(사건 진행 카드/다음 단계 안내)
            → "이어서" 또는 "다음 추천 문서" → 문서 생성

[문서 생성] 1.정보·증거 입력 → 2.초안 미리보기 → 3.수정·저장
            (반박문일 때) 1단계에 상대방 문서 분석·반박 목록 포함
            저장 전/결제 필요 시 → 결제 Modal → 생성 진입
```

## 8. Claude Code 전달용 구현 지시문 (초안)
1. `src/components/ui|document|form|payment|layout|feedback` 폴더로 컴포넌트 분리, 위 매핑표 경로 준수.
2. 디자인 토큰은 Fluent 2 변수(`fluent-tokens.css`) + 브랜드 오버라이드(`--brand-rest:#2D4DAA`, 배경 그라데이션 `#E9F6FF→#E1E3E0`)만 사용. 화면별 색/폰트/간격 임의 지정 금지.
3. 페이지는 컴포넌트 인스턴스 조합으로만 구성. 신규 버튼/입력/카드 스타일 생성 금지.
4. 문서 미리보기·결제·모달·상태(빈/로딩/에러)는 반드시 공통 컴포넌트 사용.
5. PRD v1.2에 없는 기능 추가 금지.
